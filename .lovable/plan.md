## Что делаем

Упакуем сайт rde163 как самостоятельный демо-кейс, который можно показать где угодно — на лендинге-портфолио, отправить PDF, поделиться ссылкой. Готовим три формата одновременно: интерактивный, видео и слайды.

## 1. Демо-режим (главный артефакт)

Новый маршрут `/demo` — guided-presentation, которая не зависит от домена и работает на любом хосте.

**UX:**
- Верхняя HUD-полоса: логотип кейса РДЭ, прогресс (3/12), кнопки ← / →, переключатель Desktop ↔ Mobile (iframe-обёртка с rescale), кнопки «Авто-тур» / «Пауза», «Полный экран», «Скачать PDF».
- Левый сайд: список глав (Главная → Каталог → Карточка → Корзина → Личный кабинет → Админка → Заказы → Аналитика → Мобильная версия).
- Центр: scaled iframe с реальной страницей сайта в режиме `?demo=1` (mock-данные, скрытые цены, без обращений к Supabase) + аннотации поверх — пины с подписями «персональная цена», «live-остатки», «8 складов».
- Низ: подпись главы + 1-2 строки контекста («что показываем и почему это сложно»).
- ←/→ и пробел листают, `F` — фуллскрин, `M` — мобильный вид. URL хранит шаг: `/demo/catalog`, `/demo/admin/orders`.

**Demo-mode флаг:**
- `?demo=1` или localStorage → отключает реальные запросы, подменяет данные на детерминированный mock (бренды, остатки, заказы, скидки), убирает чувствительные блоки (телефоны, договорные цены), отключает submit форм с тостом «Демо-режим».
- `SiteHeader` показывает badge «DEMO» и скрывает кнопку «Войти».

## 2. Авто-тур (видео-режим)

Та же страница `/demo`, но с кнопкой «Play tour»: автоматическое перелистывание глав с настроенными таймингами (4-6 сек на главу), плавный crossfade, ken-burns подзум на ключевых блоках. Это покрывает запрос «мини-видео, которое пролистывает все страницы» без необходимости рендерить mp4 — записывается с экрана в любой момент.

Опционально — экспорт в MP4 через Remotion (см. п. 5), если нужен файл для соцсетей.

## 3. Лендинг кейса `/portfolio/rde163`

Отдельная страница-обёртка, которую можно встроить iframe-ом на ваш будущий сайт-портфолио:
- Hero: название проекта, роль, стек (TanStack Start, React, Supabase, Tailwind), 2-3 ключевых метрики (6000+ SKU, 8 складов, ~30 сек на заказ).
- Скриншот-карусель из 6-8 ключевых экранов (embla, уже есть в проекте).
- Кнопка «Открыть интерактивное демо» → `/demo`.
- Список решённых задач: B2B-каталог, персональные скидки, live-остатки, админка с импортом, транзакционные email.
- Технические детали (стек, архитектура) — раскрывающийся блок.
- Кнопка «Скачать PDF».

## 4. PDF-вариант (handout)

Печатный режим `/demo/print` — все главы стопкой, по одной на страницу A4 landscape, с аннотациями. `Cmd+P → Save as PDF` даёт готовый файл для коммерческих предложений. Без отдельной библиотеки.

## 5. (опционально) MP4-таур

Remotion-композиция 30-40 сек, прокликивает 8 ключевых экранов с подписями и фоновой музыкой-тишиной. Рендерится через CLI в `/mnt/documents/rde163-demo.mp4`. Делаем только если подтвердите — это +существенный объём работы.

## Технические детали

```
src/routes/
  demo.tsx                 # layout: HUD + сайдбар + scaled iframe + Outlet
  demo.index.tsx           # редирект на первую главу
  demo.$chapter.tsx        # рендер главы по slug
  demo.print.tsx           # печатный режим (все главы стопкой)
  portfolio.rde163.tsx     # страница-кейс
src/demo/
  chapters.ts              # массив глав: slug, title, route, viewport, annotations[], voiceover
  mock-data.ts             # детерминированные mock-объекты для всего сайта
  DemoFrame.tsx            # iframe-обёртка со scale-to-fit + viewport switch
  Annotation.tsx           # пин с линией и подписью поверх iframe
  useDemoMode.ts           # хук-флаг (читает ?demo, localStorage, env)
```

**Demo-mode пробрасывается в существующий код через 1 хук:**
- `useDemoMode()` возвращает `{enabled, data}` — все хуки `useBrands/useProducts/useOrders` проверяют флаг и возвращают mock вместо server fn.
- Формы (поиск, submit заказа) при demo-mode показывают toast «Это демо» и не отправляют запросы.

**Что НЕ трогаем:** Supabase, реальные роуты `/catalog`, `/admin`, `/account` — продолжают работать как раньше для авторизованных пользователей. Демо — параллельный слой.

## Что вы получаете

1. Ссылка `id-preview--…lovable.app/demo` — кликабельная презентация, можно вставить в резюме / Telegram / Behance.
2. Страница `/portfolio/rde163` — самостоятельный кейс для вашего будущего сайта-портфолио (через iframe или копированием кода).
3. PDF через `Cmd+P` на `/demo/print`.
4. Видеозапись авто-тура через любую screen-recorder (или MP4 через Remotion, если подтвердите п. 5).
