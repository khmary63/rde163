# Интернет-магазин «Русский дом экспорта» (ГРОСС Запчасти)

B2B-платформа самообслуживания: каталог запчастей для китайской спецтехники, персональные цены, заявки менеджеру, личный кабинет. Цветовая палитра и стилистика — на базе gross-parts.ru (тёмный индустриальный фон + ярко-голубой акцент).

## Подход к реализации

Работаем в два больших шага по вашему выбору:

1. **Шаг A — Фронт-прототип всех ключевых страниц** на моковых данных. Дизайн, навигация, состояния — без бэкенда. На этом шаге согласуем визуал и UX.
2. **Шаг B — Подключение бизнес-логики** через Lovable Cloud (БД, авторизация, заявки, админка) поверх готового фронта.

Ниже — содержание каждого шага и итоговая архитектура.

---

## Шаг A. Фронт-прототип (текущая итерация)

### A.1. Дизайн-направления
Сначала сгенерирую **3 варианта главной страницы** (тёмная индустриальная база, голубой акцент `#00BCE4`, рубленая техническая типографика — но с разной композицией: «утилитарный B2B-терминал», «премиум-индустриальный лендинг», «динамичный с техно-акцентами»). Вы выбираете один — на нём строим всё остальное.

### A.2. Структура маршрутов (TanStack Start)

```text
/                       Главная (лендинг)
/catalog                Каталог: поиск, фильтры, список позиций, чекбоксы
/cart                   Корзина и оформление заявки
/checkout/success       «Заказ принят»
/account                Личный кабинет (layout с сайдбаром)
  /account              Профиль и моя скидка
  /account/orders       Текущие заказы
  /account/purchases    Ваши покупки (история)
  /account/templates    Шаблоны повторных заказов
  /account/documents    Документы на оплату
  /account/dashboard    Дашборд статистики
  /account/manager      Личный менеджер
/blog                   Список статей
/blog/$slug             Статья
/reviews                Отзывы
/contacts               Контакты + Яндекс.Карта
/login, /register       Авторизация (выбор «Физлицо/Юрлицо»)
/admin                  Админ-панель (layout)
  /admin/users          Пользователи и типы
  /admin/discounts      Скидки
  /admin/managers       Менеджеры
  /admin/catalog        Товары/прайс
  /admin/brands         Бренды
  /admin/warehouses     Склады
  /admin/home           Контент главной + CTA
  /admin/blog           Блог
  /admin/reviews        Отзывы
  /admin/contacts       Контакты/соцсети
  /admin/logs           Логи интеграций
```

### A.3. Ключевые UI-блоки

**Глобально:** тёмная шапка с логотипом, телефоном, мессенджерами, контактом администратора магазина; липкая поисковая строка по каталогу при скролле; виджет обратной связи (плавающая кнопка) — пока форма «оставить сообщение менеджеру».

**Главная:**
- Hero с сильным оффером и быстрым переходом в каталог.
- Сценарий «Нашёл за 30 секунд» (3 шага).
- Быстрые кнопки: «Ищу по артикулу», «Подобрать по бренду», «Помощь менеджера», «Загрузить список», «Под заказ».
- Блок «Почему нам доверяют» (наличие на складах, персональные скидки, закреплённый менеджер, документы в кабинете, повтор заказа, прямая работа по РФ).
- Преимущества, отзывы (превью), блог (превью), контакты + Яндекс.Карта.
- Все CTA-блоки: «Оформи договор сегодня», «Стать партнёром», «Нужна консультация» и т.д.

**Каталог:**
- Слева — фильтры: наименование, артикул, цена; наличие; склады (8 точек, чекбоксы, по умолчанию все включены); бренд; оригинал/аналог (по умолчанию «оригинал»).
- Справа — таблица позиций (строки): наименование, артикул, мин. характеристики, склад, наличие, **цветной бейдж статуса** (зелёный/жёлтый/красный), цена со скидкой, чекбокс, кнопка «Оформить заказ».
- Сверху — поиск + быстрые сценарные кнопки.
- Массовые действия: «Добавить выбранное в корзину», «Скачать заявку в Excel».
- Конверсионный блок «Не нашли деталь?» (артикул/файл/фото + телефон).

**Корзина:**
- Список с наименованием, артикулом, складом, количеством, ценой со скидкой, статусом наличия.
- Опции по строке: «Купить» / «Забронировать» / «Оформить под заказ» (логика выбора по статусу).
- Если позиции с разных складов — переключатель «Единым счётом / Разными счётами» с группировкой по складам и галочками.
- Итог + кнопка «Отправить заявку менеджеру».

**Личный кабинет:** все разделы из ТЗ с моковыми данными, плюс карточка персонального менеджера (ФИО, телефон, email, фото) и виджет связи.

**Админка:** базовый layout с таблицами CRUD — пока на моковых данных, чтобы согласовать структуру. Полная привязка к БД — на шаге B.

### A.4. Дизайн-токены
- Фон: глубокий тёмно-сине-чёрный `#0A0E13` / поверхности `#11161D`.
- Акцент: ярко-голубой `#00BCE4` (как на gross-parts.ru).
- Текст: белый/серый (`#E5E9F0` / `#8A95A5`).
- Бейджи статусов: зелёный `#22C55E`, жёлтый `#F59E0B`, красный `#EF4444`.
- Типографика: техническая рубленая (Oswald/Bebas для заголовков, Inter для текста).
- Все цвета — через семантические токены в `src/styles.css` (oklch).

### A.5. Моковые данные для прототипа
Один типизированный модуль `src/data/mock.ts`: ~80 позиций каталога с разными складами/статусами, 8 складов, ~10 брендов, демо-пользователь с персональной скидкой 12%, демо-менеджер, 3 статьи блога, 5 отзывов.

## Шаг B. Подключение бизнес-логики (следующая итерация)

После согласования прототипа включаем **Lovable Cloud** и реализуем:

### B.1. Авторизация и роли
- Регистрация с выбором «Физлицо / Юрлицо» (разные обязательные поля по ТЗ).
- Подтверждение email.
- Роли: `guest`, `customer_individual`, `customer_company`, `manager`, `admin` — отдельная таблица `user_roles` + security definer `has_role()` (без хранения роли в profile, против privilege escalation).

### B.2. Схема БД (основные таблицы)
- `profiles` (тип клиента, контакты, привязка к менеджеру)
- `user_roles`
- `discounts` (наименование/ИНН → процент скидки, привязка к user_id)
- `managers` (ФИО, телефон, email, фото)
- `warehouses` (8 складов из ТЗ)
- `brands`
- `products` (артикул, наименование, бренд, оригинал/аналог, характеристики)
- `stock` (product_id, warehouse_id, qty, status, expected_qty, expected_date)
- `orders`, `order_items` (со статусом «Купить/Забронировать/Под заказ» по строке, привязкой к складу и счёту)
- `order_documents` (загруженные менеджером счета/счёт-фактуры в Storage)
- `order_templates` (шаблоны для повторных заказов)
- `cta_blocks`, `home_content`, `contacts`, `social_links` (управляемый контент)
- `blog_posts`, `reviews`, `review_replies`
- `feedback_messages` (виджет связи)
- `sync_logs` (журнал обновления остатков)
- RLS на всех таблицах: пользователь видит только свои заказы/документы; админ/менеджер — всё.

### B.3. Каталог как «готовый прайс»
По вашему уточнению: данные о товарах = существующий прайс. На шаге B вы загружаете прайс (Excel/CSV) в админку → парсер сохраняет в `products` + `stock`. Архитектурно закладываем интерфейс для будущей синхронизации с Google Sheets, но MVP — ручная загрузка файла. Журнал последнего обновления и логи ошибок — в `sync_logs`.

### B.4. Бизнес-логика заказа
- Цены пересчитываются с учётом `discounts` на уровне server function (нельзя доверять клиенту).
- При «Забронировать» проверка `qty заказа vs expected_qty` — если превышает, предложение «оставшееся под заказ» (split строки).
- Группировка по складам + сценарий «единый счёт / разные счета».
- Экспорт заявки в Excel (`xlsx` библиотека на сервере): артикул, производитель, наименование, кол-во, цена, склад — отдельный файл на каждый счёт.
- Email менеджеру с прикреплённой Excel-заявкой (Lovable Email).

### B.5. Личный кабинет — функциональность
- История заказов с возможностью «открыть, отредактировать, оформить заново».
- Сохранение заказа как шаблона.
- Дашборд: количество/сумма заказов, периодичность, топ брендов/категорий/складов (server function + recharts).
- Раздел «Документы» — загрузка менеджером в Storage, скачивание клиентом.

### B.6. Админ-панель
CRUD по всем сущностям из раздела 21 ТЗ + переключение заказа в «Ваши покупки», ручные корректировки сроков поставки, просмотр логов интеграций.

### B.7. Виджет обратной связи
По вашему выбору — пока форма «оставить сообщение». Сообщения сохраняются в `feedback_messages`, уведомление на email менеджера. ИИ-сотрудник можно подключить позже через AI Gateway.

### B.8. SEO
- ЧПУ для категорий/брендов/блога.
- `head()` с уникальными meta на каждой странице (TanStack Start SSR).
- JSON-LD микроразметка для Organization, Product, BreadcrumbList, BlogPosting.
- `sitemap.xml` через server route.

### B.9. Что **сознательно вне MVP**
Будут заложены архитектурно, но реализованы позже (Этап 3 ТЗ): автоподгрузка отзывов с Яндекс.Карт, ретаргетинг-пиксели, интеграция с CRM/1С (передача данных), VIN-поиск, Google Sheets синхронизация, ИИ-сотрудник в виджете, мультибрендовые разделы.

## Технический стек
- **Фронт:** TanStack Start v1, React 19, Tailwind v4, shadcn/ui.
- **Бэк:** Lovable Cloud (Postgres + Auth + Storage + Edge serverless), server functions с `requireSupabaseAuth`, server routes для webhook-ов.
- **Доп. библиотеки (на шаге B):** `xlsx` для Excel-экспорта, `recharts` для дашборда, `react-markdown` для блога.
