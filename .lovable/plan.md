## Что меняем

Каталог собирается из двух источников:

1. **Excel-прайс** (загрузка в админке) — основной ассортимент с ценами и остатками на 8 складах.
2. **Google Sheets** (автосинхронизация) — позиции «под заказ», которых нет на складах.

Цена, которую видит покупатель в каталоге, выбирается по его персональной скидке из профиля (`profiles.discount_percent`).

## 2. Изменения в БД (одна миграция)

В таблицу `products` добавляем:

- `price_retail numeric(12,2)` — РРЦ РДЭ (колонка 18 прайса). Используется как «без скидки».
- `price_tiers jsonb` — `{ "5": …, "10": …, "15": …, "18": …, "20": …, "21": … }`. NULL, если позиция пришла только из Google Sheets.
- `source text` — `'price_list' | 'on_order'`. Помечает, откуда последний раз обновлялся товар (для отчётности и логики «под заказ»).

Существующее `base_price` оставляем для обратной совместимости (= `price_retail`).

В `stock` ничего не меняем. Для позиций «под заказ» (Google Sheets) остаток пишется в виртуальный склад `OFFER` со статусом `expected` — добавим запись в `warehouses` (`is_public = false`, чтобы не показывать в фильтре).

## 3. Загрузка Excel в админке (`/admin/catalog`)

Полностью заменяем нынешний CSV-парсер:

- Парсер ищет лист, читает заголовки из строки 3, данные с строки 8.
- Маппинг колонок жёсткий, как в таблице выше; склад ↔ код БД — по точному названию.
- Превью: первые 10 строк, счётчики, найденные тиеры скидок, найденные склады, предупреждения по пустым/битым строкам.
- Импорт:
  1. Upsert брендов.
  2. Upsert `products` по `(brand_id, sku)`: `name`, `base_price = price_retail = col18`, `price_tiers = {…}`, `source = 'price_list'`.
  3. Полная перезапись `stock` для 8 складов из файла (удалить, вставить заново — как в текущем Google Sheets sync, но только для этих 8 складов; данные `OFFER` не трогаем).
- В UI отказываемся от шаблонов CSV/XLSX и переписываем секцию «Формат файла» под новый прайс. Снимаем поддержку `.csv` (формат фиксирован).

## 4. Google Sheets sync — позиции «под заказ»

В `src/lib/catalog-sync.server.ts`:

- Сохраняем чтение того же листа `Запчасти`.
- Колонка `J` («Цена, РРЦ») → `price_retail` и `base_price`. `price_tiers` для таких товаров считаем по формуле от РРЦ (5/10/15/18/20/21%) — округление до 2 знаков. Это позволяет применять персональную скидку и к «под заказ».
- Все строки сохраняются как `source = 'on_order'`. Если SKU уже есть с `source = 'price_list'` — обновляем только `name`, цены не трогаем.
- Остатки кладутся в виртуальный склад `OFFER` со статусом `expected` (qty = `K — свободно` или 1, если нет данных). Реальные склады (8 шт.) этим синком не затрагиваются.
- В сводке логов отдельно показываем «под заказ: N».

## 5. Цена в каталоге по персональной скидке

`src/routes/catalog.tsx`:

- Подтягиваем `discount_percent` текущего пользователя один раз (`useQuery` по `profiles`). Анонимам — 0%.
- В запрос `products` добавляем `price_retail, price_tiers, source`.
- Для каждой строки выбираем цену:
  1. Если `price_tiers[discount]` есть — берём его.
  2. Иначе — ближайший меньший тиер из `{5,10,15,18,20,21}`.
  3. Иначе — `price_retail`.
- В колонке «Цена» показываем выбранную цену; подпись «с учётом скидки X%» или «РРЦ», если скидки нет. Для гостей — РРЦ + подсказка «войдите, чтобы увидеть вашу цену».
- Карточки `source = 'on_order'` без остатков получают бейдж «Под заказ» и кнопку «Заказать» (уже есть ветка `status === "out"`).

## 6. Что не трогаем

- Структура `discounts` / `profiles.discount_percent` — остаётся.
- Корзина, заказы, кросс-номера, поиск.
- `admin.discounts` — управление персональными скидками работает как сейчас.

## Технические детали

### Миграция

```sql
alter table public.products
  add column price_retail numeric(12,2) not null default 0,
  add column price_tiers  jsonb         not null default '{}'::jsonb,
  add column source       text          not null default 'price_list';

insert into public.warehouses (code, name, city, is_active, is_public, sort_order)
values ('OFFER', 'Под заказ', null, true, false, 99)
on conflict (code) do nothing;
```

### Парсинг Excel в браузере

Используется уже установленный `xlsx`. Читаем как `sheet_to_json({ header: 1 })`, берём 3-ю строку как заголовки, со 8-й — данные, маппинг по индексу + по точному названию для складов (для устойчивости к перестановке).

### Структура `price_tiers`

```ts
type PriceTiers = Partial<Record<"5" | "10" | "15" | "18" | "20" | "21", number>>;
```

### Выбор цены клиентом

```ts
function pickPrice(p: { price_retail: number; price_tiers: PriceTiers }, discount: number): number {
  const tiers = [5, 10, 15, 18, 20, 21] as const;
  const exact = p.price_tiers[String(discount) as keyof PriceTiers];
  if (exact != null) return exact;
  const lower = [...tiers].reverse().find((t) => t <= discount);
  return lower != null ? (p.price_tiers[String(lower) as keyof PriceTiers] ?? p.price_retail) : p.price_retail;
}
```

- `supabase/migrations/<new>.sql` — миграция.
- `src/lib/catalog-sync.server.ts` — переписать на `source='on_order'` + виртуальный склад.
- `src/routes/admin.catalog.tsx` — новый парсер Excel под фиксированный формат, новая секция «Формат файла», удаление CSV-шаблонов.
- `src/routes/catalog.tsx` — загрузка `discount_percent`, выбор цены по тиерам, корректные подписи.
